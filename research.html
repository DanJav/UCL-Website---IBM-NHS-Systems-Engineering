Technology review

React

A key part of the project is altering Jitsi’s UI, for which we needed to choose a JavaScript framework or library. The two most widely used libraries/frameworks in this field are React, which is an open-source library, and Angular, which is a framework. For this project, we have decided to keep using React. routes and scalability.

Advantages of React

The main advantage of React is that it’s smaller in size than Angular, which is very popular web framework. This means that on average less Javascript code has to be sent and run by the client, which makes a noticebale performance difference for devices with small memory sizes or slower internet connection. React is also easier to start with if the programmer has prior knowledge of JavaScript as it relies on common Javascript patterns, oposed to Angular which introduces a lot of complex desgins such as dependency injection which may prove unnecessary for medium sized projects. React provides enough advantages to make the achievement of our project requirements due to its ease of use, all while providing the tools necessary to complete the project due to its compatibility with third party modules.

React also has a virtual DOM (Document Object Model) implementation. The virtual DOM holds the reactive state of the components that gets managed at Javascript runtime, which is different form the actual rendering time of the DOM. This limits the frequency of updates to the actual DOM, making it faster and more suitable for a video calling application that requires high responsiveness and less processing overhead for the devices it runs on. app.

React is very useful at improving development efficiency. Firstly, React components give the project extensibility by allowing developers to reuse code in different context. This improves readability and code quality overall as a tested and functional component can be reused in multiple parts of the project with the confidence of not introducing bugs. Also, this gives structure to complex web code bases wthat tend to be written in a very procedural manner.

The large toolset of 3rd party libraries available for React is very helpful when developing large projects. Since React is a JavaScript library it allows the usage of most vanilla Javascript plugins which reduces development time, allowing developers to focus on new features rather than recreating already available functionality.

Overall, React is the more realistic option, due to the afore mentioned arguments, accessibility and use of plain JavaScript.

Next.js

Next.js is a framework built on top of React that will empower us to create an even more performant and reactive application while reducing the complexity of the code. Some of the most important features of Next.js include automatic bundling using webpack, code-splitting, page-based routing, server-side rendering, API routes and scalability.

Performance

Performance is one of the most important factors for our choice. React requires difficult techniques for optimization. Next.js simplifies this process.

Automatic code-splitting is a Next.js feature that dramatically improves loading times of pages by dynamically loading JavaScript code that is packaged into bundles. With Next.js’s file system based routing we are free to scale our application while affecting the load times of our pages. When a specific route is requested by the client, Next.js will load that specific JavaScript bundle that corresponds to the requested route, thus not overloading the client with unused code. These bundles are automatically created by Next.js using webpack which includes optimization features by determining which code is shared across certain components.

One other feature that improves the loading time of the application is dynamic import. Certain libraries may be needed only when a certain action is done by the user, therefore Next.js supports dynamic import which can load JavaScript code when the user triggers certain actions, such as the click of a button.

Pre-rendering is what makes a difference when using Next.js with React. By default, Next.js pre-renders every page. This means that Next.js generates HTML for each page in advance, instead of having it all done by client-side JavaScript. Thus, this is solves the previously mentioned disadvantage of using React for static-generated pages. Moreover, pre-rendering can result in better performance and SEO.

Each generated HTML is associated with minimal JavaScript code necessary for that page. When a page is loaded by the browser, its JavaScript code runs and makes the page fully interactive. Next.js has two forms of pre-rendering: Static Generation and Server-side Rendering . The difference is in when it generates the HTML for a page. Static Generation generates HTML build time reusing it for each request, while the other generates HTML on each request. likely to lose support from their company.

Another important aspect of this framework is the caching mechanism for remote fetches. Vercel, the creator of Next.js provides a very lightweight library, named SWR, that improves the fetching mechanism in React. The name “SWR” is derived from stale-while-revalidate, a HTTP cache invalidation strategy popularized by HTTP RFC 5861. SWR is a strategy to first return the data from cache (stale), then send the fetch request (revalidate), and finally come with the up-to-date data. In the context of our project, SWR server a very important role in adapting the application to the Progressive Web Application (PWA) standard that would allow it to run "offline" (without the use of internet services). This enables us to create a PWA that can be installed added to devices' home screens even if it is a web app.

Deployment

Since our base code has already been designed to function well with the Vercel deployment system (as recommended by Next.js on their website), Next.js makes a great choice. The process is very easy requiring only the repository of the Next.js app and the configuration data to be copied to Vercel. This will facilitate our production testing iterations by reducing the time taken to redeploy and reconfigure our Next.js application for production. Easy to deploy using Vercel, the creator of Next.js. But gives the option of deploying on every server that has Node.js, thus allowing us to fulfil our requirement of deploying on a remote server for users to be served the application seamlessly.

Docker

Docker is a platform that enables your application to be deployed, shipped and run on different systems and environments without changing the source code. This is achieved by isolating the app from the environment of the machine that is running on (host machine). Those isolated environments are called containers which can run quickly, securely and reliably with all the needed dependencies. Docker is the industry standard for container-based applications.

Containers can be built from Docker images which represent immutable snapshots of software packages such as Ubuntu, Node.js and many others. Thus, a container can be constructed to run multiple software packages from multiple images without needing to install them on the host machine.

Another very important feature is the Docker Compose API. Docker Compose allows multiple containers to communicate at runtime, thus giving the possibility of running parts of the application from multiple environments. Docker inserts the containers in their own local network that runs on the Docker Host, separate from the network of the host machine. This enables an application to have a server container that communicates with a client container in a secure, isolated environment.

Deployment

In the context of our project, Docker is very useful in achieving cross-platform functionality. More exactly, Docker would allow clients to deploy the application consisting of the client-side, server-side and database locally on the consumer device without detriment to the features. Moreover, Docker containers are very lightweight as they share the machine’s OS system kernel and therefore do not require an OS per application. This allows our application to run on lower-end hardware and devices without consuming resources unnecessarily.

Privacy & Security

Since Docker enables our application to run entirely locally, the communication between client-server and server-database in the cloud is removed. In addition, Docker containers are very secure as they operate within their own isolated environment and network. Most important security features include: containers have their own namespace in the kernel which means docker containers cannot see and interact with host machine processes, limted attack surface of the Docker Engine and restricted root access of the container processes on host machine.
