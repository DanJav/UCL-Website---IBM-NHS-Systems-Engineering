Research

As our task is to build a social video calling platform, we have conducted research on existing similiar applications and what technologies we believe are needed for our purposes. We have had discussions with the client to determine what similiar applications and technologies we should investigate.

Similiar applications

Houseparty

Houseparty is a free video social networking app that allows you to video chat with up to 8 friends in “rooms” and play games together. Its popularity has exploded during the covid-19 crisis.

Good features
  • A wide variety of in apps games such as Trivia, UNO, Quick Draw!, etc.
  • Colourful, friendly and easy-to-navigate menu.
  • Amazing, suggestive icons.
  • In app chat.
  • Great app layout for mobile.
  • Easily add friends from contact list.
Limiting features
  • Change background feature.
  • Maximum of 8 users per room.
  • [NEED TO ADD]
Bad features
  • Connectivity not so great.
  • Video and audio quality fluctuate too much.
  • Requires log-in.
    Take-aways
  1. Games should be easily available from a drop-down menu.
  2. A key usability aspect of games is the in-synch feature for all users => We need to minimize delay between what users see.
  3. Games seem to be a little bit overcomplicated. It is not clear in certain situations whose turn it is or how should the game be played => Make games intuitive, add tooltips when the user seems lost (i.e., lack of input), easy interface for games is important as well.
  4. Icons are custom, very suggestive but their uniqueness can be a bit confusing => our app should have more commonly used SVGs for the vital functionalities.
  5. Responsiveness is a key aspect.
  6. Easy to add friends, we should try to further simplify this process.
  7. Easy to connect with contacts from other applications such as Snapchat, Whatsapp, etc. It is a very important feature => somehow allow users to connect with email and whatsapp contacts at least: could use Whatsapp API:
  8. Icons should be suggestive and easy to see.
  9. Main menu should include the vital operations of usability.
  10. We need a bit more spark of colour in the UI like Houseparty has. Our app is not for business use => Users should derive joy visually.
  11. Ability to change the background is not so great which makes the feature unusable => We should strike for something better, similar to Snapchat’s.
  12. Number of users per room is a bit low for an app designed for group of friends.
  13. No scheduled invitations, which is a must-do.

    Take-aways
  1. Very easy to contact friends (synchronized with Facebook contacts) => should include the ability to synch Facebook contacts as it is a very used and easy to log with (optional for users to do so).
  2. Room Topic is a great idea = ability to add the topic and reason of the meeting users create => add room category as part of the “Create a room” process.
  3. Start time for a meeting is a great feature. Makes the scheduling system easier than in Zoom for example. Very easy to set the “Meeting time” to “Now” in case user gets lost!
  4. All meeting settings for a “Create Meeting” process appear on the same pop-up which makes it very clear from the beginnings what the steps for setting up a meeting are => should use this as well!
  5. Emogis as SVGs is great and very intuitive for room topic and other, as they are commonly used and understood by a large group of people (even elderly understand emogis).
  6. For every popup there exists a “Go back”, “Undo” and “Cancel” button that is highlighted in grey making it very easy to see => more generally we should add navigation buttons to all popovers and highlight them.
  7. Transparent design for buttons which gives depth to the design. Also make it easier to see the “clickability” of a button. .
